2. Why Tokens and Embeddings Are Important
LLMs don't understand raw text. They work with numbers.
Text (human-readable)
"The weather is absolutely beautiful today!"
⬇
Numbers (model-readable)
[5021, 1037, 318, 8725, …]
⬇
Embedding Vector (example)
0.12   -0.34   0.56   …   -0.08   0.91
💡 Key Idea:
Tokens turn text into manageable units. Embeddings turn tokens into meaningful vectors.
3. Tokens: The Building Blocks of Text for LLMs
A token can be:
  • A whole word: "hello"
  • A subword: "##ing"
  • A character: "a"
  • A punctuation: "!"
Models operate on sequences of token IDs (integers), not text.
Example:
Text: "Playing football is fun!"
Playing
football
is
fun
!
Token IDs (example):
3456   7892   2003   4567   23   33
The same text can be tokenized differently depending on the tokenizer.
4. How Tokenization Works
Tokenization = process of breaking text into tokens.
Goals:
  • Cover all possible text
  • Be efficient (fewer tokens is better)
  • Handle rare/unknown words
  • Preserve meaning as much as possible
General Steps:
  1. 1 Normalize text (lowercase, remove extra spaces, etc.)
  2. 2 Split text (using rules or algorithms)
  3. 3 Convert tokens to token IDs (using a vocabulary)
5. Types of Tokenizers
① Whitespace Tokenizer (Simplest)
Splits on spaces.
Example: "hello world" → ["hello", "world"]
✅ Easy to implement
❌ Large vocab, can't handle rare words well
② Character Tokenizer
Splits into characters.
Example: "Hi!" → ["H", "i", "!"]
✅ Small vocab   ❌ Long sequences
③ Subword Tokenizer (Most common in LLMs)
Splits words into subword units.
Example: "unbelievable" → ["un", "believe", "able"]
✅ Handles rare words   ✅ Efficient
✅ Widely used (BPE, WordPiece)
6. Subword Tokenization in Detail
Subword tokenization balances the best of both worlds: It reduces vocabulary size while handling rare and unseen words.
1. BPE (Byte Pair Encoding)
Starts with characters. Merges most frequent pairs iteratively.
Used by: GPT-2, GPT-3, LLaMA 2
Example (low → high merges):
1. low est → low_est
2. low_est → lower
3. lower est → lowest
low
est
→
lower
est
2. WordPiece
Greedy algorithm that maximises the likelihood of the corpus.
Used by: BERT, RoBERTa
Example:
unbelievable → un##believe##able
un
##believe
##able
3. SentencePiece
Language-independent (no pre-tokenization needed).
Used by: T5, mT5, LLaMA 2
Example:
international → ▁international
▁international
▁ indicates a new word boundary

9. From Tokens to Embeddings
Each token ID is mapped to a dense vector (embedding).
Example:
TokenToken IDEmbedding (dimension = 6)
The464[0.21, -0.31, 0.45, 0.12, 0.08, -0.15]
smartest9281[0.33, -0.12, 0.77, -0.31, -0.22, …]
AI1423[0.11, 0.44, -0.33, 0.66, 0.91, …]
Embeddings capture semantic meaning in a continuous vector space.
10. How Embeddings Are Created
Embeddings are learned during training. Two main methods:
1. Lookup Embeddings (Most common)
  • Each token ID looks up a vector in an embedding matrix.
  • Matrix is learned during training.
Example:
Token ID → Embedding Vector (dim = 6)
464 → [0.12, -0.34, …, 0.91]
2. Contextual Embeddings (Advanced)
  • Embedding depends on context.
  • Generated by models like BERT, GPT.
  • Same token can have different embeddings in different sentences.
11. Types of Embeddings
1. Static Embeddings (Non-contextual)
Same for a word regardless of context.
Example: word2vec
2. Contextual Embeddings
Change based on surrounding words.
Example: BERT
3. Positional Embeddings
Encode position in the sequence.
Used in Transformers (Positional Encoding)
12. Visualizing Embeddings
Embeddings live in high-dimensional space. We can visualize them using dimensionality reduction (e.g., t-SNE, PCA).
king 👑 queen 👸 man 👨 woman 👩
apple 🍎 banana 🍌 car 🚗 truck 🚛 bus 🚌
Words with similar meanings are closer in vector space!
Transformers = Token Embeddings + Positional Embeddings
13. Properties of Good Embeddings
  • Capture semantic similarity
  • Generalize to unseen data
  • Smooth and continuous space
  • Support mathematical operations
king − man + woman = queen
This is possible because embeddings capture relationships mathematically!
14. Challenges & Limitations
  • High-dimensional vectors (costly to compute)
  • Curse of dimensionality
  • Bias in embeddings (can reflect societal biases)
  • OOV (Out-of-Vocabulary) words in static embeddings
  • Contextual embeddings are computationally expensive

15. Tokenization in Practice (Python Example)

16. Embedding in Practice (Python Example)
Code Example:
from transformers import AutoTokenizer, AutoModel
import torch

# Load model and tokenizer
tokenizer = AutoTokenizer.from_pretrained(
    "bert-base-uncased")
model = AutoModel.from_pretrained(
    "bert-base-uncased")

inputs = tokenizer(texts, padding=True,
    truncation=True, return_tensors='pt')

# Get embeddings (last hidden state)
with torch.no_grad():
    outputs = model(**inputs)
    embeddings = outputs.last_hidden_state

print(embeddings.shape)
# (1, seq_len, hidden_dim)
Used in Transformers to get contextual embeddings.
Example Output:
embeddings.shape
torch.Size([1, 10, 768])
1 = batch size
10 = sequence length
768 = embedding dimension (hidden size)
Simple API, powerful embeddings.
17. Key Takeaways
  • Tokens convert text into model-understandable units.
  • Tokenization affects efficiency and performance.
  • Subword tokenization (BPE, WordPiece) is the standard in modern LLMs.
  • Embeddings turn tokens into meaningful vectors.
  • Embeddings can be static, contextual, or positional.
  • Embeddings capture semantic relationships in continuous space.
  • They are the foundation for nearly all LLM capabilities.
