3. Model I/O: Loading Quantized Models (GGUF)
Quantization = reducing the number of bits used to represent model weights → lower VRAM, faster inference, slight loss in precision.
3.1 What is Quantization?
Fewer bits = smaller size & faster, but less precise.
Float 32-bit
3.141592741022573Z
→ High Precision
Float 16-bit
3.140625
→ Lower Precision
3.2 Load Phi-3 (GGUF) with LangChain (llama-cpp-python)
from langchain import LlamaCpp

llm = LlamaCpp(
  model_path="Phi-3-mini-4k-instruct-fp16.gguf",
  n_gpu_layers=-1,  # use all GPU layers
  n_ctx=2048,         # context window
  max_tokens=500,
  seed=42,
  verbose=False
)

llm.invoke("Hi! My name is Maarten. What is 1 + 1?")

5. Controlling Model Output
5.1 Next Token Generation
LLM assigns a probability to all possible next tokens.
Prompt: "I am driving a …"
TokenProbability
car0.45
truck0.30
bike0.15
elephant0.01
……
Highest probability token is chosen (unless sampling is enabled).
5.2 Temperature (Randomness)
Controls creativity. Higher = more random.
Low Temp (0.2)
Low → deterministic (predictable)
High Temp (1.0)
High → diverse (creative)
car   truck   bike   elephant
5.3 Top-p (Nucleus Sampling)
Considers tokens whose cumulative probability ≥ top_p.
Low top_p (0.2)
Few tokens considered
High top_p (1.0)
Many tokens considered
Low → fewer tokens considered
High → more tokens considered
5.4 Choosing Parameters (Examples)
Use CaseTemperatureTop-pDescription
Brainstorming / IdeasHigh (0.8–1.2)High (0.8–1.0)Max creativity & diversity
Creative WritingHigh (0.7–1.0)Low (0.2–0.7)Creative but still coherent
Email / ReportsLow (0.1–0.3)Low (0.1–0.5)Focused, accurate, predictable
TranslationLow (0.1–0.3)High (0.8–1.0)Stable with vocab variety
Code GenerationLow (0.2–0.4)Low (0.1–0.5)Accurate, consistent code
💡 Tip: Lower temperature & top_p → more predictable. Higher values → more creativity & diversity.
6. Intro to Prompt Engineering
Prompt engineering = designing inputs to guide the LLM to produce useful, accurate, and safe outputs.
The Basic Ingredients of a Prompt
  • 🎭 Persona — Define the role the LLM should take.
  • 📋 Instruction — What task or question to perform.
  • 🌍 Context — Background information / why.
  • 📄 Format — Output format (JSON, list, bullets, etc.).
  • 👥 Audience — Who is the target audience?
  • 🎵 Tone — Style of response (formal, friendly, etc.).
  • 📊 Data — Main input or text to process.
  • 🎯 Output Indicators — Guide output (e.g., "Sentiment:").

  • 🏷️ Classification — "Classify the sentiment of this review."
  • 📝 Summarization — "Summarize the following article."
  • 🔍 Information Extraction — "Extract all dates and locations."

6.3 In-Context Learning (Provide Examples)
Types
  • Zero-shot — No examples provided.
  • One-shot — One example provided.
  • Few-shot — Two or more examples provided.
Example (One-shot)
User: A "Gigamuru" is a type of Japanese musical instrument. An example of a sentence that uses the word Gigamuru is:
Assistant: I have a Gigamuru that my uncle gave me as a gift. I love to play it at home.

User: To "screeg" something is to swing a sword at it. An example of a sentence that uses the word screeg is:
Assistant: <model generates answer>
7. Advanced Prompting Techniques
7.1 Chain-of-Thought (CoT)
Ask the model to "think" before answering. Zero-shot CoT example:
Q: The cafeteria had 23 apples. If they used 20 to make lunch and bought 6 more, how many apples do they have? Let's think step-by-step.

A:
Step 1: Start with 23 apples.
Step 2: Used 20 → 23 − 20 = 3
Step 3: Bought 6 → 3 + 6 = 9
Answer: 9 apples.
Helps with math, logic, reasoning, and multi-step tasks.
7.2 Self-Consistency (SC)
Sample multiple reasoning paths & take majority answer.
Path 1
Ans: 9
Path 2
Ans: 10
Path 3
Ans: 9
Majority Vote
Final Answer: 9
  • Improves accuracy
  • More compute (multiple calls)
  • Best used with CoT
7.3 Tree-of-Thought (ToT)
Explore multiple reasoning paths as a tree.
Question
Thought 1
Thought 2
Thought 3
✅ ✅ ❌
  • Explore many ideas
  • Prune low-quality thoughts
  • Great for creative & complex tasks
8. Output Verification
Ensure outputs are structured, valid, safe, and accurate.
  • Structured Output — Use examples, grammars, or tools (JSON, lists, etc.).
  • Valid Output — Constrain output to allowed values or formats.
  • Ethics & Safety — Filter profanity, PII, bias, and harmful content.
  • Accuracy — Verify facts, reduce hallucinations.
How to Control Output
  1. 1Provide Examples (few-shot learning)
  2. 2Grammar / Constrained Sampling (e.g., JSON grammar)
  3. 3Fine-tuning (discussed in Chapter 12)
9. Memory: Helping LLMs Remember Conversations
9.1 Conversation Buffer
(Remember everything)
2 3 4 5 n=3
Stores entire chat history.
  • No information lost
  • Consumes many tokens
9.2 Window Buffer
(Remember last k turns)
Keeps only last k interactions.
  • Lower token usage
  • Older info forgotten
9.3 Summary Memory
(Summarize history)
Summarizes full history using another LLM.
  • Lower tokens
  • Long conversations possible
  • Extra LLM call for summary
Memory is added to the prompt so the LLM can use past context.
Example Flow:
User says…
→
Memory updates
→
Prompt + History
→
LLM Responds
