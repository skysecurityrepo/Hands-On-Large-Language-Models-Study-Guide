7.2 Layer Norm (At a Glance)
Stabilizes training by normalizing across features.
For a vector x = {x₁, x₂, …, xₙ}:

LayerNorm(x) = γ · x − μ / √(σ² + ε) + β
  • μ = mean(x), σ² = variance(x)
  • γ (scale) and β (shift) are learned parameters.

8. Attention Is All You Need
Transformers rely on attention to mix information across the sequence.
Two main steps:
  1. 1
    Score how relevant each previous token is to the current token.
  2. 2
    Use those scores to combine information from all positions into a single output vector.
Attention allows the model to focus on the most relevant parts of the context.

10. Recent Improvements to the Transformer Architecture
10.1 More Effective Attention: Local / Sparse Attention
  • Full attention is expensive for long sequences.
  • Sparse attention attends to a subset of tokens.
  • Local attention only attends to nearby tokens.
  • Reduces compute and memory.
  • Konstantina (more accuracy).
Full Attention   Local Attention   Sparse Attention
10.2 Multi-Query & Group-Query Attention
  • Multi-Head Attention (MHA): each head has its own K and V.
  • Multi-Query (MQA): all heads share the same K and V (only Q differs).
  • Group-Query (GQA): heads are grouped, each group shares K and V.
  • MQA/GQA reduce memory bandwidth and speed up inference.
10.3 Optimizing Attention: From MHA → MQA → GQA

14. Flash Attention
  • Standard attention reads/writes large matrices to memory.
  • FlashAttention computes attention in blocks, minimizing memory access.
  • Much faster and memory efficient, especially across domains.

12. Positional Embedding: RoPE (Rotary)
  • Adds position information via rotation in query and key space.
  • Preserves relative positions.
  • Enables extrapolation to longer sequences.
  • Common in modern LLMs.
R(θ) = [ cos θ   -sin θ ]
        [ sin θ    cos θ ]

Q' = R(θ)Q,   K' = R(θ)K
13. Other Architecture Experiments
  • Many variants exist beyond the original Transformer.
  • Examples: Reformer, Performer, Linformer, RETRO, Hyena, etc.
  • Explore different ways to scale attention, reduce cost, and improve efficiency.
  • Research is ongoing and evolving quickly.
14. Transformers Beyond LLMs
  • Vision Transformers (ViT) for images.
  • Audio Transformers for speech.
  • Time-series forecasting.
  • Code models and multimodal models.
  • Transformers are a general-purpose architecture across domains.

  • Better attention mechanisms

🔑 Key Takeaways
  • Transformers use self-attention to understand context.
  • Attention scores relevance, then combines information.
  • Efficient tricks (caching, MQA/GQA, FlashAttention) make generation fast.
  • Positional encoding (e.g., RoPE) adds order information.
  • Many improvements and variants continue to push the limits.
  • Transformers power not just LLMs, but many AI applications.
