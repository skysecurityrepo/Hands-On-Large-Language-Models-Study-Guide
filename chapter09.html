Cross-Encoder (BERT) vs Bi-Encoder (SBERT)
Cross-Encoder / BERT
Slow but accurate
Sentence A + Sentence B → BERT + Classifier Head → Score
Bi-Encoder / SBERT (Siamese)
Fast & scalable
Sentence A → BERT (shared weights) → Pooling (mean / [CLS])
Sentence B → BERT (shared weights) → Pooling (mean / [CLS])
→ Cosine Similarity
Bi-encoders scale to millions of comparisons!

Benefits
  • Reduces hallucinations
  • More factual & up-to-date
  • Explainable answers
Challenges
  • Retriever quality matters
  • Chunking & routing complexity
  • Evaluation is harder

  • Query Rewriting
    Rewrite user question for better retrieval
  • Multi-Query RAG
    Break into multiple sub-queries
  • Multi-Hop RAG
    Sequential searches for complex Qs
  • Query Routing
    Route to best data source / tool
  • Agentic RAG
    Use agents to decide actions/tools
16. Chunking Strategies
How to split long documents before embedding.
  • By Sentence
    Fine-grained; may lose context
  • By Paragraph
    Good balance for many docs
  • Fixed-size (N tokens)
    Simple; may cut meaning
  • Overlapping Chunks
    Retain context across boundaries
  • Semantic Chunking (LLM)
    Split by meaning/structure

19. Best Practices & Tips
  • Choose strong embedding models
  • Use hybrid search when possible
  • Rerank shortlists (not full corpus)
  • Craft good prompts for RAG
  • Cite sources for transparency
  • Evaluate with multiple metrics
  • Monitor & iterate continuously
