1. What is Prompt Engineering?
  • The art and science of crafting inputs (prompts) to guide LLMs toward desired outputs.
  • Improves relevance, accuracy, consistency, and safety of responses.
  • Enables complex reasoning, structured output, and reliable automation.
  • Includes prompt design, iterative refinement, verification, and evaluation.

3. Loading a Text Generation Model (Example: Phi-3-Mini)
Using transformers pipeline:
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer,
                           pipeline

model = AutoModelForCausalLM.from_pretrained(
    "microsoft/Phi-3-mini-4k-instruct",
    device_map="cuda",
    torch_dtype="auto",
    trust_remote_code=True
)

tokenizer = AutoTokenizer.from_pretrained(
    "microsoft/Phi-3-mini-4k-instruct")

pipe = pipeline(
    "text-generation",
    model=model,
    tokenizer=tokenizer,
    return_full_text=False,
    max_new_tokens=500,
    do_sample=False  # deterministic output
)
pipeline handles tokenization, chat template, and generation for you.

5.1 Temperature (Controls Randomness)
Higher temperature = more random/creative
Lower temperature = more deterministic
Effect of Temperature
Low Temp (0.2)
car   truck   bike   elephant
Low → deterministic (predictable)
High Temp (1.0)
car   truck   bike   elephant
High → diverse (creative)
output = pipe(messages, do_sample=True,
  temperature=1.6)
# Every run may produce different output!
5.2 Top-p (Nucleus Sampling)
Considers tokens whose cumulative probability >= top_p.
Low top_p (0.2)
consider fewer tokens
Only top tokens sampled
High top_p (1.0)
consider more tokens
Wider token sampling 
output = pipe(messages, do_sample=True,
  top_p=0.9)
5.3 Choosing Parameters (Examples)

6. The Basic Ingredients of a Prompt
Prompt engineering = designing inputs to guide the LLM to produce useful, accurate, and safe outputs.
The Basic Ingredients of a Prompt
  • 🎭 Persona — Define the role the LLM should take on.
  • 📋 Instruction — The task or question to be completed.
  • 🌍 Context — Background information to help the model.
  • 📄 Format — How the output should be structured.
  • 👥 Audience — Who is the response for? (e.g., experts, kids)
  • 🎵 Tone — Style of response (formal, friendly, etc.).
  • 📊 Data — Main input or text to be processed.
  • 🎯 Output Indicators — Guide output (e.g., "Answer:", JSON).

11. Reasoning with Generative Models
Mimic human System 2 thinking (slow, logical, reflective).
System 1 Thinking (Intuitive)
  • Automatic
  • Pattern matching
  • No reflection
  • More errors
System 2 Thinking (Logical)
  • Step-by-step
  • Reflective
  • More reliable
  • Better for complex tasks
Techniques to enable reasoning:
  • Chain-of-Thought (CoT)
  • Self-Consistency
  • Tree-of-Thought (ToT)
  • Verification & Evaluation
12. Chain-of-Thought (CoT)
Encourage the model to "think" before answering.
Example Zero-shot CoT:
User: The cafeteria had 23 apples. If they used 20 to make lunch and bought 6 more, how many apples do they have? Let's think step-by-step.
Step 1: Start with 23 apples.
Step 2: Used 20 → 23 − 20 = 3 apples left.
Step 3: Bought 6 more → 3 + 6 = 9 apples.
Answer: 9 apples.
Helps with math, logic, reasoning, and multi-step tasks.
13. Self-Consistency (SC)
Sample multiple reasoning paths & take majority answer.
Same Question
Path 1
Ans: 9
Path 2
Ans: 10
Path N
Ans: 9
Majority Vote
Final Answer: 9
  • Improves accuracy
  • More compute (multiple calls)
  • Best used with CoT

Example: Allowed outputs → {"positive", "neutral", "negative"}

With Grammar (Constrained)

18. Practical Python Example (Granular Control)
Using llama-cpp-python with JSON grammar
from llama_cpp import Llama

llm = Llama.from_pretrained(
    repo_id="microsoft/Phi-3-mini-4k-instruct-gguf",
    filename="*fp16.gguf", n_gpu_layers=-1, n_ctx=2048
)

output = llm.create_chat_completion(
  messages=[{"role": "user",
    "content": "Create a warrior for an RPG game in JSON format."}],
  response_format={"type": "json_object"},
)

print(json.dumps(json.loads(
  output["choices"][0]["message"]["content"]),
  indent=4))
19. Best Practices & Tips
  • Be specific & clear in instructions.
  • Provide context and examples.
  • Use CoT for complex tasks.
  • Control randomness for consistency.
  • Track experiments & results.
  • Validate critical outputs.
