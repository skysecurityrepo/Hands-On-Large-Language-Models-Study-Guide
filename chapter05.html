CHAPTER 5: TEXT CLUSTERING & TOPIC MODELING

Discovering structure and themes in large collections of text without labels. We combine embedding models, dimensionality reduction, clustering, and representation models (classic + generative) to cluster documents and extract meaningful topics.

🧠
1. What is Text Clustering?
Group semantically similar texts based on meaning and relationships.
  • Efficiently organize large volumes of unstructured text
  • Enables exploratory data analysis
  • Find outliers, speed up labeling, and detect mislabeled data
  • Works without supervision
Example:
Cluster ArXiv papers to discover hidden research themes.
2. What is Topic Modeling?
Discover (abstract) topics in a collection of documents.
  • Topics are described by keywords or keyphrases
  • Each topic has one overarching label (usually)
  • Helps summarize large corpora and understand main themes
Example Topic:
Topic label: Neural Machine Translation
Keywords: translation, nmt, machine, neural, seqseq, attention, bleu

4. Common Pipeline for Text Clustering
A widely used 3-step pipeline:
  1. 1
    Embed Documents
    Convert text → embeddings using an embedding model.
  2. 2
    Reduce Dimensionality
    Compress embeddings to a lower-dimensional space (e.g., UMAP).
  3. 3
    Cluster Documents
    Find groups of similar documents (e.g., HDBSCAN).
① Embedding Documents
Convert documents to dense vector representations that capture semantic meaning.
Model Example: thenlper/gte-small
from sentence_transformers import SentenceTransformer

model = SentenceTransformer("thenlper/gte-small")
embeddings = model.encode(abstracts, show_progress_bar=True)
Output: Each document → 384-dimensional vector

Use embedding models optimized for semantic similarity. Good embeddings = better clusters.
② Reduce Dimensionality (UMAP)
High-dimensional data is hard to cluster. UMAP preserves global structure in fewer dimensions.
from umap import UMAP

umap_model = UMAP(n_components=5, min_dist=0.0,
                  metric="cosine", random_state=42)
reduced_embeddings = umap_model.fit_transform(embeddings)

Why UMAP?
  • Handles nonlinear structures well & keeps clusters tight.
  • n_components: 5–10 works well in practice.
③ Cluster Reduced Embeddings (HDBSCAN)
Density-based clustering. Finds clusters of any shape. Automatically detects outliers.
from hdbscan import HDBSCAN

hdbscan_model = HDBSCAN(min_cluster_size=50,
    cluster_selection_method="eom")
clusters = hdbscan_model.fit_predict(reduced_embeddings)
Result:
Cluster 0 Cluster 1 Cluster 2 Cluster 3 … Cluster 155 Outliers (-1)
Lower min_cluster_size → more clusters. Outliers (-1) are points that don't belong to any cluster.

15. Challenges & Tips
Challenges
  • Choosing the right number of components
  • Outliers & noise
  • Redundant keywords
  • Interpreting topic quality
  • Large-scale computation
Tips
  • Use semantic embeddings optimized for similarity
  • Validate topics manually
  • Use representation models for better labels
  • Experiment with parameters
16. Key Takeaways
  • Text clustering groups similar documents without labels.
  • Topic modeling extracts themes described by keywords or labels.
  • BERTopic combines clustering + c-TF-IDF + representation models.
  • Modular design lets you mix & match the best models.
  • Representation models (KeyBERT, MMR, LLMs) greatly improve topics.
  • Visualization + inspection = meaningful, interpretable topics.
17. Glossary
TermDefinition
EmbeddingDense vector representation of text.
UMAPDimensionality reduction preserving structure.
HDBSCANDensity-based clustering algorithm.
OutlierPoint not assigned to any cluster.
c-TF-IDFCluster Term Frequency (word count per cluster), modified Inverse Document Frequency emphasizing unique words.
TopicA discovered theme in a collection of documents.
Representation ModelModel that refines topic keywords or labels.
