2. Overview: 3 Major Categories
Modern search systems powered by language models.
🔵
Dense Retrieval
Convert query & docs to embeddings and retrieve nearest neighbors.
🔀
Reranking
Take top-k results and reorder them by relevance using an LLM.
🤖
RAG (Retrieval-Augmented Generation)
Retrieve relevant info and pass it to an LLM to generate a grounded answer (with citations).

5.2 Pipeline
How dense retrieval works.
  1. Split documents into chunks
  2. Embed chunks & store in vector database / index
  3. Embed query
  4. Find nearest neighbors (similarity search)
  5. Return top-k results

Example (Cohere RAG)
results = co.rerank(query=query, documents=texts, top_n=3,
                   return_documents=True)
docs = [{"text": doc.document.text} for doc in results.results]
response = co.chat(message=query, documents=docs)

8. Chunking & Indexing
Chunking Strategies
  • By sentence
  • By paragraph
  • Fixed-size (3–8 sentences)
  • Overlapping (retain context)
Indexing Options
  • NumPy (small corpora)
  • ANN libraries (FAISS, Annoy)
  • Vector databases (Weaviate, Pinecone)
Tips
✅ Smaller chunks → more precise results
✅ Larger chunks → more context
✅ Overlapping improves recall
✅ Choose based on your data & queries
9. Advanced RAG Techniques
9.1 Query Rewriting
Rewrite queries to improve retrieval.
User: "We have an essay due tomorrow… Where do dolphins live for example?"
⬇
Rewritten: "Where do dolphins live?"
9.2 Multi-Hop RAG
Generate multiple queries.
User: "Compare car manufacturers in 2003? Do they each do nocs? Make EVs or not?"

Step 1: "largest car manufacturers 2023"
→ Toyota, Volkswagen, Hyundai

Step 2: "Toyota electric vehicles"
Step 2: "Volkswagen electric vehicles"
Step 2: "Hyundai electric vehicles"
9.4 Query Routing
Route to right data source.
Q about HR →
Search HR system (Notion)
Q about customers →
Search CRM (Salesforce)
Q about docs →
Search internal docs

10.2 RAG Evaluation (RAGAS Metrics)
🔵 Faithfulness
Answer consistent with context
🔵 Answer Relevance
How relevant is the answer?
🔵 Context Precision
How precise is retrieved context?
🔵 Context Recall
Is all relevant info retrieved?
🔵 Answer Correctness
Is the answer factually correct?
Human evaluation = gold standard. LLM-as-a-Judge can automate.
11. Hands-On Code Example: Local RAG with LangChain
Step 1: Load Models
from langchain import LlamaCpp
from langchain import LlamaCpp
import HuggingFaceEmbeddings

llm = LlamaCpp(
  model_path="Phi-3-mini-4k-instruct-fp16.gguf",
  n_gpu_layers=1, max_tokens=500, n_ctx=2048
)

embeddings = HuggingFaceEmbeddings(
  model_name="thenlper/gte-small"
)
Step 2: Build Vector Index
from langchain.vectorstores import FAISS

texts = [...]  # list of document chunks

db = FAISS.from_texts(texts, embeddings)
retriever = db.as_retriever(search_kwargs={
  "k": 2  # top 3 results
})
Step 3: Prompt Template
template = """[User]
Relevant Information:
{context}

Provide a concise answer to the
following question using the
relevant information above:
{question}[end]>
[assistant]"""

prompt = PromptTemplate(
  template=template,
  input_variables=["context", "question"]
)
Step 4: RAG Chain
from langchain.chains import RetrievalQA

rag = RetrievalQA.from_chain_type(
  llm=llm,
  chain_type="stuff",
  retriever=retriever,
  return_source_documents=True,
  chain_type_kwargs={"prompt": prompt}
)
